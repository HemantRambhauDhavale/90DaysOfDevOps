# Day 39 – CI/CD Basics

Today I learned the concepts of CI/CD.

## Why CI/CD Exists

- Manual deployments create mistakes
- Environment mismatch causes issues
- Slow release process

---

## Continuous Integration (CI)

- Developers merge code frequently
- Automated build and tests run on every push
- Helps catch issues early

---

## Continuous Delivery

- Code stays deployment-ready
- Manual approval before production release

---

## Continuous Deployment

- Code automatically deploys after passing tests
- No manual approval required

---

## Pipeline Anatomy

- Trigger → starts pipeline
- Stage → build/test/deploy phase
- Job → unit of work
- Step → command inside job
- Runner → machine executing pipeline
- Artifact → output files/images

---

## Summary

Today I learned:

- Why CI/CD is important
- Difference between CI / Delivery / Deployment
- Basic pipeline terminology
- Importance of automation in DevOps
